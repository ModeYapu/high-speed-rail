# 🎵 音效系统测试指南

## 快速测试

1. **启动游戏**
   ```bash
   cd /root/.openclaw/workspace/high-speed-rail
   python3 -m http.server 8080
   ```
   访问：http://localhost:8080

2. **选择列车和模式**
   - 选择和谐号或复兴号
   - 选择"自由驾驶"模式

3. **测试音效**

### 引擎声
- 按 **W** 或 **↑** 加速
- 引擎声会随速度变化（低沉→尖锐）
- 速度越快，音调越高

### 刹车声
- 按 **S** 或 **↓** 制动
- 会听到金属摩擦声
- 紧急制动：按 **空格键**

### 鸣笛声
- 点击右侧控制面板的 **🔔 鸣笛** 按钮
- 会听到高铁特色的鸣笛声

### 风声
- 加速到 50km/h 以上
- 会听到呼啸的风声
- 速度越快，风声越大

### 雨声
- 选择"恶劣天气"模式
- 会听到淅沥的雨声
- 雨声强度与天气效果同步

### 音效控制
- 点击右侧控制面板的 **🔊 音效** 按钮
- 可以切换静音/开启音效

## 音效清单

| 音效 | 触发条件 | 持续时间 | 特点 |
|------|---------|---------|------|
| 引擎声 | 速度 > 5km/h | 持续 | 频率随速度变化 |
| 刹车声 | 制动时 | 1.5秒 | 金属摩擦效果 |
| 鸣笛声 | 点击按钮 | 2秒 | 大三度和弦 |

| 车站广播 | 到达终点 | 0.5秒 | 三音提示 |

## 技术细节

### 音效实现
- **技术**：Web Audio API
- **类型**：纯合成音效（无外部文件）
- **优势**：
  - 文件体积小
  - 可实时调整
  - 性能优秀

### 性能优化
- 引擎声只在需要时启动
- 停止时立即释放资源
- 音量平滑过渡避免爆音
- 使用低开销的振荡器组合

### 常见问题

**Q: 听不到音效？**
- 检查浏览器音量
- 检查系统音量
- 确认音效按钮未静音
- 点击页面激活音频上下文

**Q: 音效有延迟？**
- 正常现象，Web Audio API需要用户交互才能启动
- 首次播放可能有50-100ms延迟

**Q: 音效卡顿？**
- 关闭其他占用CPU的程序
- 检查浏览器性能
